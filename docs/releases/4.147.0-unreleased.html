Version 4.147.0

Upcoming release · In development · Not yet available on NuGet

Highlights

This upcoming release expands the HarfBuzz API surface with a new HBColor struct and fills in missing wrappers across the binding layer, while also hardening native dependency versions (libjpeg-turbo 3.1.4.1, freetype 2.14.3, zlib 1.3.2.1). WebAssembly support is streamlined by dropping pre-.NET 8 Emscripten targets, and ❤️ @ebariche cut SkiaFiddle's WASM bundle size by ~60%.

Breaking Changes

None in this release.

New Features

API Surface

  • HBColor struct — Adds a proper HBColor value type that correctly represents hb_color_t, replacing raw integer usage in HarfBuzz color APIs. (#4000)
  • Missing API wrappers — Fills in several overlooked HarfBuzz and SkiaSharp wrappers and corrects naming-convention violations throughout the binding layer. (#4001)

WebAssembly

  • Drop pre-.NET 8 Emscripten targets — Native WASM builds for Emscripten versions prior to .NET 8 are removed, simplifying the build matrix and reducing maintenance overhead. (#4022)
  • SkiaFiddle bundle size reduced ~60% — Significant reduction in the SkiaFiddle WASM application size through dependency trimming and build optimisation. ❤️ @ebariche (#3849)

Platform

  • Tizen SDK updated to 10.0 — The Tizen build toolchain is updated from SDK 6.1 to 10.0, enabling support for current Tizen targets. (#3743)
  • Fixed Uno Platform Gallery sample references — Corrects stale SkiaSharp package references in the Uno Platform Gallery sample project. ❤️ @ramezgerges (#3867)

Security / Dependency Updates

  • libjpeg-turbo 3.1.4.1 — Updates the bundled libjpeg-turbo to the latest patch release. (#4012)
  • freetype 2.14.3 — Updates FreeType to address upstream bug fixes and security patches. (#3726)
  • zlib 1.3.2.1-motley — Updates zlib to the latest motley fork revision. (#3720)
